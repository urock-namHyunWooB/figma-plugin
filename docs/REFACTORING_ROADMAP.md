# 리팩토링 로드맵

> 다양한 UI 대응을 위한 인터페이스/구조체/구현체 분리

## 최종 목표

**현재:** Figma → React (Emotion/Tailwind) 전용 컴파일러
**목표:** 다양한 UI 프레임워크로 확장 가능한 구조

---

## 핵심 문제

1. **거대한 클래스** - `_TempAstTree` (1,646줄), `_FinalAstTree` (3,497줄)에 변환 로직 집중
2. **하드코딩된 파이프라인** - 단계 추가/변경 시 여러 파일 수정 필요
3. **테스트 어려움** - 개별 변환 단계 단위 테스트 불가
4. **확장성 부족** - 새 출력 형식 추가 시 대규모 수정 필요

---

## 리팩토링 방향

### Phase 1: 파이프라인 추상화
- 20개 이상의 변환 단계를 독립적인 단위로 분리
- 각 단계별 테스트 가능하게
- 변환 순서 유연하게 변경 가능하게

### Phase 2: Manager 책임 분리
- 대형 Manager 클래스들 역할별로 분해
- 단일 책임 원칙 적용

### Phase 3: 확장성 개선
- 새 기능 추가 시 기존 코드 수정 최소화
- 출력 형식별 Strategy 분리

### Phase 4: 타입 안전성 강화
- `any` 타입 제거
- 명확한 타입 정의
